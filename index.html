What's is IEX?

IEX is an improved version of ie9.js created by Dean Edwards. Add support for DOM2, addEventListener, placeHolder, inline-block and most important full PNG for IE6. AlphaImageloder is replaced by VML because it brought problems to the performance of the browser, as well as supporting bicubic images and background-repeat as background-position.

Benefits

  • VML Fix for IE6- PNG
  • Fix Hover on IE6-
  • Focus and active on IE7-
  • Box Model fix on IE5/IE6 Quirks
  • inline-block support for IE7-
  • querySelector support for IE7-
  • addEventListener support for IE8-
  • HTML5 tags support
  • Opacity Support
  • placeHover Support on input and textarea
  • Save 85% of IE adapting codes
  • Make IE little faster and stable

Setup and to consider

Internet Explorer 5.5

Not every configuration and/or copy of Internet Explorer 5.5 works with IEXfix. Service Pack 2 and KB2962872 are required.

IE5.x comes with very weak VML/VGX support, which will not allow transparent PNGs to work, so upgrade to IE6's VGX.

Internet Explorer 6

Internet Explorer 6 can work in both Normal and Quirks modes. The original IE9.js worked in quirks mode only with IE5.5. But IEX activates quirks mode as long as «document.compatMode != 'CSS1Compat'».

We recommend using Quirks mode for IE6 since Normal mode doesn't support «box-sizing: bordex-box;»

Internet Explorer 7

Internet Explorer 7 can work in both Normal and Quirks modes. The original IE9.js worked in quirks mode only with IE5.5. But IEX activates quirks mode as long as «document.compatMode != 'CSS1Compat'».

We recommend using Quirks mode for IE7 since Normal mode doesn't support «box-sizing: bordex-box;»

Internet Explorer 8

You can use normal mode without problems as it supports «box-sizing: bordex-box;». Although with <meta http-equiv="X-UA-Compatible" content="IE=5"> you can use the quirks mode without problems.

Internet Explorer 9

You can use normal mode as it supports «box-sizing: bordex-box;», but IE9 has certain limitations. We recommend using IE8 mode with <meta http-equiv="X-UA-Compatible" content="IE=8">, or quirks mode with <meta http-equiv="X-UA-Compatible" content="IE=5">.
